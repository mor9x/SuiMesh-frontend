## 一、产品背景与目标

本项目基于 Figma 社区设计稿「Battlefield 4 - Main Menu UI Design (Community)」进行高保真交互原型开发，目标是以纯前端（HTML/CSS/JS）无依赖方式，还原 BF4 主菜单的完整视觉语言与交互流程，供教学、展示、作品集使用，并通过 GitHub Pages 对外可访问。

**核心目标：**
- 视觉 100% 忠实 Figma 设计稿颜色/字体/布局系统
- 全流程可操作：从主菜单 → 注册 → 选世界 → 组队 → 开房间
- 任何人打开链接即可"玩"（无需安装/登录）
- 代码结构清晰，可二次开发

## 三、功能模块

### 3.1 启动序列（Boot Screen）
- 全屏黑底 + 系统初始化文本逐行打印（模拟 HUD 启动）
- 进度条加载至 100% 后自动进入主菜单
- 时长约 2.5 秒，可按任意键跳过

### 3.2 主菜单（Main Menu）
- **左侧导航栏**：8 个模式入口（Multiplayer、Campaign、Co-Op、Soldier、World、Invite、Room、Register），悬停高亮 + 橙色激活条
- **右侧详情面板**：选中模式后展示说明文字、子选项 Grid（3×2）、CTA 按钮、4 项统计数据
- **顶部 Header**：BATTLEFIELD 4 Logo、实时在线人数（每 4 秒随机波动）、时钟、快捷标签页
- **底部 Footer**：玩家档案卡、XP 进度条、新闻滚动条（每 6 秒切换）、快捷操作按钮

### 3.3 注册页（Register）
- 表单字段：名/姓、Soldier Tag、Email、密码确认、服务器区域选择
- 前端验证：必填项、Email 格式、密码强度提示、Tag 唯一性模拟
- 提交后跳转至世界选择页

### 3.4 世界与身份（World & Identity）
- **左半：** 6 个战区卡片（上海/昆仑/新加坡/南海/阿塞拜疆/海南），点击选中 + 状态标签（ACTIVE / CONTESTED / HOSTILE）
- **右半：** 4 种兵种角色卡（突击/工程/支援/侦察），图标 + 描述 + 选中高亮；Callsign 输入框
- 底部 CTA：确认选择，进入组队页

### 3.5 邀请好友（Invite Friends）
- 搜索框实时过滤好友列表
- 好友卡片：头像首字母、名称、在线状态、兵种、段位
- 邀请最多 3 人组队；右侧小队槽位实时更新；可移除
- 底部：生成房间邀请链接（复制到剪贴板）

### 3.6 开房间（Open Room）
- 左侧配置区：房间名/最大人数/区域/可见性/游戏模式切换/票数/车辆延迟/玩法开关滑块
- 右侧：地图轮转预览（前/后翻页）+ 当前玩家列表（Ping 指示/踢人）
- 底部：「LAUNCH ROOM」按钮 → 加载态 → 成功提示

## 五、技术要求

- **纯单文件**：`index.html`（HTML + CSS + JS 内联），零依赖，零构建步骤
- **外部资源**：仅 Google Fonts（CDN）
- **兼容性**：Chrome 100+、Safari 15+、Firefox 100+
- **性能**：首屏加载 < 1s（无外部图片）；Canvas 动画帧率 ≥ 55fps
- **部署**：GitHub Pages（main 分支），URL 直接可访问
